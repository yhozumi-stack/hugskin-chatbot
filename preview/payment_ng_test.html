与信NGリカバリーの再現ページ

テストモード(URLパラメータ):
・?ng=1 … 与信NG画面(パスワードは質問シナリオ) → 案内→PW再質問→カード→送信
・?ng=1&skip=1 … skip:{password:'auto'} のLP → 案内→カードのみ→送信(PWは自動生成)
・?ng=1&nopw=1 … フォームにPW欄が無いLP → 案内→カードのみ→送信
・?ng=1&held=1 … 引き継ぎ成功ケース → 案内→カードのみ→送信(PW=held-pass-1234が自動復元)
・?ng=1&pm=21 … クレカ自体の失敗(pm=クレカID) → 発動しない(通常フロー)
・パラメータ無し … 通常LP。チャットを完走すると自動送信がこの?ng=1画面へ遷移(実際の弾き返しを一気通貫で再現。PW引き継ぎの本番同等テスト)
・?ng=1&off=1 … paymentFallbackタグ未設定 → 何も起きない(既定OFFの確認)

20 : 与信審査が通りませんでした。大変恐れいりますが他の決済手段をご利用ください。
