# HugSkin チャットボット タグ設定 変更メモ

このメモは、ecforceタグ管理に貼るタグのうち、下の部分を変えるための早見表です。

```js
window.HS_CHAT = {
  // ここに設定を書く
};
```

基本ルール:
- LPのHTMLは触らず、`window.HS_CHAT` の中だけを変更する。
- 各設定行の末尾には `,` を付ける。
- `chatbot.js` 本体を変えた時だけ、GitHubへpushして `<script src="...chatbot.js?v=13">` の `v=` を上げる。
- タグ内の設定だけを変える場合は、基本的に `chatbot.js` のpushは不要。ecforceタグ管理の内容を差し替える。

---

## 初回自動起動について

初回にチャットを自動で開かない:

```js
autoOpen: 'manual',                    /* 初回自動起動なし */
```

初回にチャットをすぐ開く:

```js
autoOpen: 'immediate',                 /* 初回自動起動あり。ページ表示後すぐ開く */
```

ページを30%スクロールしたら開く:

```js
autoOpen: 'scroll30',                  /* 30%スクロールで自動起動 */
```

数字を変えるとスクロール率が変わる:

```js
autoOpen: 'scroll50',                  /* 50%スクロールで自動起動 */
```

5秒後に開く:

```js
autoOpen: 'delay5000',                 /* 5000ミリ秒後に自動起動 */
```

注意:
- `autoOpen` は「勝手に開くかどうか」の設定。
- 右下のフローティングバナーを出す/消す設定は `launcher`。

---

## CTAボタンでチャットを開く

LP内のCTAボタンを押した時にチャットを開く:

```js
openTriggers: 'a[href="#lp-form"]',    /* CTAボタンでチャットを開く */
```

HugSkinのLPはCTAリンクが `href="#lp-form"` になっていることが多いので、通常はこのままでOK。

注意:
- `launcher: false` かつ `autoOpen: 'manual'` にする場合、`openTriggers` がないとチャットを開く手段がなくなる。

---

## フローティングバナーについて

右下のフローティングバナーを消す:

```js
launcher: false,                       /* フローティングバナーを非表示 */
```

右下のフローティングバナーを出す:

```js
launcher: true,                        /* フローティングバナーを表示 */
```

バナーの文言を変える:

```js
launcher: true,
launcherText: 'かんたん注文はこちら',
```

画像バナーにする:

```js
launcher: true,
launcherImage: 'https://yhozumi-stack.github.io/hugskin-chatbot/img/banner.png',
```

注意:
- `launcher: false` はバナーだけを消す。チャット本体の表示形式は `mode: 'float'` のままでOK。
- フローティングバナーを消しても、`autoOpen: 'immediate'` なら初回は自動で開く。

---

## よく使う起動パターン

初回は自動で開く、フローティングバナーは消す、閉じた後はCTAで開く:

```js
mode: 'float',
autoOpen: 'immediate',
openTriggers: 'a[href="#lp-form"]',
launcher: false,
```

初回は自動で開かない、フローティングバナーも消す、CTAを押した時だけ開く:

```js
mode: 'float',
autoOpen: 'manual',
openTriggers: 'a[href="#lp-form"]',
launcher: false,
```

30%スクロールで開く、フローティングバナーは消す、CTAでも開ける:

```js
mode: 'float',
autoOpen: 'scroll30',
openTriggers: 'a[href="#lp-form"]',
launcher: false,
```

フローティングバナーを出して、初回は自動で開かない:

```js
mode: 'float',
autoOpen: 'manual',
launcher: true,
launcherText: 'かんたん注文はこちら',
```

## シナリオを変える

form-plus再現シナリオにする:

```js
scenario: 'formplus',
```

通常シナリオにする:

```js
scenario: 'standard',
```

冒頭で会員かどうかを聞くシナリオにする:

```js
scenario: 'member_ask',
```

注意:
- 質問の中身そのものを大きく変える場合は `chatbot.js` の `SCENARIOS` 側を編集する。
- LPごとにシナリオを切り替えるだけならタグ側の `scenario` でOK。

---

## 価格・商品名・オファー文言

価格をタグ側で指定する:

```js
vars: {
  PRICE: '2,980円（税込）',
},
```

商品名も指定したい場合:

```js
vars: {
  PRODUCT: 'HugSkin',
  PRICE: '2,980円（税込）',
  OFFER: '初回限定',
},
```

注意:
- `PRODUCT` を空にすると、LP内の商品名から自動取得する。
- `PRICE` はLPから自動取得しない。挨拶文などに価格を出したい時だけ指定する。
- form-plusシナリオでは、現在の挨拶で `PRICE` を使っていないことがある。その場合は残していても表示には影響しない。

---

## 挨拶文を変える

冒頭の挨拶をタグ側で上書きする:

```js
greeting: 'こんにちは！\nかんたん注文チャットです。\nただいま{{PRICE}}でご案内中です。',
```

注意:
- 改行は `\n` と書く。
- `{{PRICE}}` などの変数は `vars` の値に置き換わる。
- 変数が空の場合、その変数を含む行は丸ごと非表示になる。

---

## 各質問の文言を変える

質問文だけをタグ側で上書きする:

```js
texts: {
  name: 'まず、お名前を教えてください',
  birthdate: '次に生年月日と性別をご入力ください',
  addr: 'お届け先と電話番号をご入力ください',
  contact: 'メールアドレスとパスワードをご入力ください',
  payment: '最後にお支払い方法を選択してください',
},
```

form-plusシナリオでよく使うキー:
- `name`: 名前、フリガナ
- `birthdate`: 生年月日、性別
- `addr`: 住所、電話番号
- `contact`: メールアドレス、パスワード
- `payment`: 支払い方法
- `card`: クレジットカード情報

standardシナリオでよく使うキー:
- `name`: 名前、フリガナ
- `contact`: メールアドレス、電話番号
- `password`: パスワード
- `sex`: 性別
- `birthdate`: 生年月日
- `zip`: 郵便番号
- `addr`: 住所
- `payment`: 支払い方法
- `card`: クレジットカード情報

注意:
- `texts` で変えられるのは、主に質問の冒頭文。
- 入力欄のラベルやplaceholder、選択肢そのものを変えたい場合は `chatbot.js` の `SCENARIOS` 編集が必要。

---

## 質問の順番を変える

form-plusシナリオで順番を変える例:

```js
stepOrder: ['name', 'addr', 'birthdate', 'contact', 'payment'],
```

standardシナリオで順番を変える例:

```js
stepOrder: ['name', 'zip', 'addr', 'birthdate', 'contact', 'password', 'payment'],
```

注意:
- 書かなかった質問は、元の順番のまま後ろに続く。
- カード入力は支払い方法の直後、確認画面は最後に自動で配置される。

---

## 質問をスキップする

パスワードを聞かずに自動生成する:

```js
skip: {
  password: 'auto',
},
```

性別を聞かずに固定する:

```js
skip: {
  sex: '2',
},
```

生年月日を聞かずに固定する:

```js
skip: {
  birthdate: '1990/01/01',
},
```

メールを聞かずに自動生成する:

```js
skip: {
  email: 'auto',
},
```

支払い方法を固定する:

```js
skip: {
  payment: 'credit_card',
},
```

複数まとめてスキップする:

```js
skip: {
  password: 'auto',
  birthdate: '1990/01/01',
  sex: '2',
},
```

注意:
- スキップした項目はチャット上で質問されず、転記時に裏側で値が入る。
- `email` を固定値でスキップするのはNG。同一顧客扱いになる恐れがあるため、`auto` か `{RAND}` 入りにする。
- `tel` のダミー値は後払い審査に影響する可能性がある。
- `password: 'auto'` にすると、購入者がマイページに入りづらくなる可能性がある。

## 冒頭画像を変える

冒頭画像を1枚出す:

```js
opening: {
  image: 'https://yhozumi-stack.github.io/hugskin-chatbot/img/chat_fv2_2980.jpeg',
  stockCheck: true,
},
```

冒頭画像を複数枚出す:

```js
opening: {
  images: [
    'https://yhozumi-stack.github.io/hugskin-chatbot/img/chat_fv1_6980.png',
    'https://yhozumi-stack.github.io/hugskin-chatbot/img/chat_fv2_2980.jpeg',
  ],
  stockCheck: true,
},
```

冒頭画像を出さない:

```js
opening: {
  image: '',
  stockCheck: true,
},
```

在庫確認風の演出を消す:

```js
opening: {
  stockCheck: false,
},
```

注意:
- `images` がある場合は複数枚表示。
- `image` は1枚だけ表示。

## カウントダウンバー

カウントダウンバーを出す:

```js
countdown: true,
```

文言を変える:

```js
countdown: {
  text: '本日受付終了まで',
},
```

カウントダウンバーを出さない:

```js

```

注意:
- カウントダウンは毎日23:59:59までの残り時間。

## 確認画面

確認画面をモーダル表示にする:

```js
summaryOptions: {
  modal: true,
},
```

確認画面の前に一言出す:

```js
summaryOptions: {
  msg: '内容をご確認ください',
},
```

確定ボタンの文言を変える:

```js
summaryOptions: {
  submitLabel: 'この内容で注文を確定する',
},
```

商品・金額ブロックを消す:

```js
summaryOptions: {
  showOrderInfo: false,
},
```

特商法・定期条件の注意文を消す:

```js
summaryOptions: {
  showLaw: false,
},
```

複数まとめて指定する:

```js
summaryOptions: {
  modal: true,
  msg: '',
  submitLabel: 'この内容で注文を確定する',
},
```

注意:
- 確認画面スキップ運用では、商品・金額・注意文を消すかどうかは慎重に確認する。

## 利用規約チェック

利用規約チェックを表示し、チェック済みで開始:

```js
summaryOptions: {
  agree: true,
},
```

利用規約チェックを表示し、未チェックで開始:

```js
summaryOptions: {
  agree: 'unchecked',
},
```

利用規約チェックを非表示:

```js
summaryOptions: {
  agree: false,
},
```

同意文言を変える:

```js
agreeText: '利用規約に同意して申し込みます。',
```

利用規約リンクを変える:

```js
agreeLink: 'https://hugskin.shop/info/customer_term',
```

リンクを消す:

```js
agreeLink: '',
```

## 支払い方法の文言

支払いボタンの表示文言を変える:

```js
paymentLabels: {
  'クレジット': 'クレジットカード【手数料0円】',
  '後払い': 'ベリトランス後払い（コンビニ後払い）',
},
```

注意:
- 変更後の文言にも「クレジット」「後払い」の語を残す。
- その語が消えると、カード入力や後払い規約表示の判定に影響する。

## 注文ボタンの自動クリック

チャット完了時にLP側の注文ボタンを自動で押す:

```js
autoSubmit: true,
```

チャット完了時に自動で押さない:

```js
autoSubmit: false,
```

注意:
- LPが「確認画面を表示しない」設定の場合、`autoSubmit: true` だと注文確定まで進む可能性がある。
- テスト時や慎重に確認したいLPでは `autoSubmit: false` も検討する。

## 表示モード

通常のフローティングチャット:

```js
mode: 'float',
```

LP内の指定場所に埋め込む:

```js
mode: 'inline',
inlineSelector: '#hs-chat',
```

注意:
- 通常運用は `mode: 'float'`。
- `inline` はLP内に `#hs-chat` のような設置場所が必要。

## 反映とキャッシュ

タグだけを変えた場合:
- ecforceタグ管理に貼っているタグを差し替える。
- `chatbot.js?v=13` の `v=` は基本そのままでOK。

`chatbot.js` を変えた場合:
- GitHubへpushする。
- 反映させたいLPのタグで `chatbot.js?v=13` の数字を上げる。

例:

```html
<script src="https://yhozumi-stack.github.io/hugskin-chatbot/chatbot.js?v=14" defer></script>
```

注意:
- `v=` を上げないと、ブラウザキャッシュで古い `chatbot.js` が動くことがある。

---

## 最低限の確認リスト

タグを変えたら確認すること:
- 初回に開く/開かないが意図通りか。
- フローティングバナーが出る/出ないが意図通りか。
- CTAボタンでチャットが開くか。
- 最後まで入力できるか。
- 確認画面に商品・金額・注意文が必要通り出ているか。
- 注文フォームに値が正しく反映されるか。
- `autoSubmit` の挙動がLPの運用と合っているか。
