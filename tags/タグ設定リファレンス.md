# タグ設定リファレンス — タグだけで制御できること全一覧

**このページの全項目は「ecforce管理画面 > タグ管理」のタグを編集するだけで効く。push不要・即時反映・そのLPだけに効く。**
チャットボットは `window.HS_CHAT = { ... }` の中に、離脱ポップアップは `window.HS_POPUP = { ... }` の中に書く。
書かなかった項目はすべて既定値で動く(既定値は chatbot.js / popup.js の①DEFAULTSが正本)。

---

## 1. チャットボット (window.HS_CHAT)

### 1-1. 起動タイミング

```js
autoOpen: 'immediate',   // ON: LP表示と同時に自動で開く(既定)
autoOpen: 'scroll30',    // ON: 30%スクロールしたら開く(数字は変更可。scroll50なら50%)
autoOpen: 'delay3000',   // ON: 3秒後に開く(ミリ秒指定。delay10000なら10秒)
autoOpen: 'manual',      // OFF: 自動では開かない(ランチャーやCTAから開く)
```

### 1-2. LP内のCTAボタンで開く(LPのHTMLは触らない)

```js
openTriggers: 'a[href="#lp-form"]',   // ON: このセレクタに合うボタンのクリックで開く(HugSkinのLPはCTAが全部このアンカー)
openTriggers: '',                     // OFF(既定)
```

### 1-3. フローティングバナー(ランチャー)

```js
launcher: true,                          // ON(既定): 右下にバナーを出す
launcher: false,                         // OFF: バナーを消す(⚠️autoOpenかopenTriggersのどちらかは残すこと。両方ないと開く手段が無くなる)
launcherText: '🎁 初回1,980円で試す',     // 文言を変える
launcherImage: 'https://…/banner.png',   // 文字の代わりに画像バナーにする
```

### 1-4. 冒頭の演出

```js
opening: {
  image: 'https://…/opening.png',      // ON: 冒頭に画像を出す('' で出さない)
  images: ['url1', 'url2'],            // ON: 複数枚を上から順に出す
  stockCheck: true,                    // ON(既定): 「ご案内枠を確認中…」演出
  stockCheck: false,                   // OFF: 演出なしで即開始
  stockGifUrl: 'https://…/stock.gif',  // 演出をGIF画像にする('' ならスピナー)
  stockTextChecking: '🔍 ご案内枠を確認しています…',   // 演出中の文言
  stockTextDone: '✅ ご案内枠を確保しました！…',        // 演出後の文言
},
```

### 1-5. 文言・変数(シナリオを触らずに変える)

```js
vars: { PRODUCT: '商品名', PRICE: '初回1,980円(税込)', OFFER: '初回限定' },
    // シナリオ文中の {{PRODUCT}} {{PRICE}} {{OFFER}} に差し込まれる。
    // ⚠️{{VAR}}を含む行は、その変数が未設定なら「行ごと」自動非表示(文言が壊れない)
greeting: 'こんにちは！✨\nいまだけ{{PRICE}}です🎁',   // 挨拶文の上書き('' ならシナリオ既定)
texts: { name: 'お名前をどうぞ！', zip: '郵便番号を教えてください' },
    // 各質問文の上書き(キー=ステップkey: name/contact/password/sex/birthdate/zip/addr/payment/card)
stepImages: { payment: 'https://…/payment_promo.png' },   // その質問の直前に画像を出す
fieldNotes: { email: '入力欄の下に出る注意書き' },          // 入力欄下の注意書き(キー=項目key)
codNotice: '利用規約\n\n…',   // 後払い選択直後に出す規約文('' で非表示)

/* --- 後払い選択直後の表示は2パターン選べる(v3.26.1〜) --- */
// パターンA: バナーだけ(Befas/I-ne方式)
codNoticeImage: 'https://yhozumi-stack.github.io/hugskin-chatbot/img/np_wiz_banner.png',
codNotice: '',
// パターンB: バナー+規約文(hideForm利用LPはこちら推奨。規約の表示場所がチャットしか無いため)
codNoticeImage: 'https://yhozumi-stack.github.io/hugskin-chatbot/img/np_wiz_banner.png',
codNotice: 'NP後払いwizの規約文…',
// どちらも書かない = 既定(規約文のみ・バナーなし)
```

### 1-6. シナリオの切替・質問の並べ替え

```js
scenario: 'standard',      // 使うシナリオ(既定)。'formplus' / 'member_ask' / 自作の別名シナリオ名
stepOrder: ['name', 'addr', 'birthdate', 'contact', 'payment'],
    // 質問の順番をタグから並べ替え(カード入力は支払い直後、確認画面は最後に自動配置)
```

### 1-7. 質問を減らす(skip=聞かずにダミー値で転記)

```js
skip: {
  password:  'auto',                  // ランダム生成
  email:     'auto',                  // 毎回ユニークなダミー(⚠️固定値は禁止。同一顧客に束ねられ事故る)
  email:     'guest+{RAND}@hugskin.jp',  // {RAND}=ユニーク文字列に置換
  tel:       '09000000000',           // 固定値(⚠️後払いLPでは与信に影響するのでtelは聞くこと)
  birthdate: '1990/01/01',
  sex:       '2',
  payment:   '102',                   // 選択肢ステップも消せる(値=そのLPのpayment_method_id)
},
// skipに入れた項目は質問から自動で消える。戻したければ行を消すだけ。
// ⚠️emailダミー=注文確認メール不達・カゴ落ちメール不達 / passwordダミー=マイページ不可。レシピ9の注意を必ず読む
```

### 1-8. カード入力・CVV

```js
cardCvv: 'auto',    // 既定: LPフォームにCVV欄があれば聞く・無ければ聞かない
cardCvv: true,      // ON: 常に聞く
cardCvv: false,     // OFF: 常に聞かない(⚠️HugSkin本番LPは全タグこれが必須。ecforceがCVV欄を非表示で残すため'auto'だと聞いてしまう)
```

### 1-9. 確認画面(summary)

```js
summaryOptions: {
  msg: 'お支払いは【後払い】で承ります✨\nご注文内容をご確認ください',   // 確認画面前の口上(既定は口上なし)
  submitLabel: '注文する →',        // 確定ボタンの文言
  showOrderInfo: false,             // OFF: ご注文内容(商品・金額)の自動表示を消す(既定はON)
  showLaw: false,                   // OFF: 特商法・定期条件(qa-caution)の表示を消す(既定はON)
  modal: true,                      // ON: form-plus風のモーダルで確認画面を出す(既定はチャット内カード)
  agree: true,                      // 既定: 同意チェックを表示・チェック済みで開始
  agree: 'unchecked',               // 表示・未チェックで開始
  agree: false,                     // OFF: 同意チェック非表示
  agreeText: '利用規約に同意して申し込みます。…',   // 同意文言の変更
  agreeLink: 'https://…/customer_term',            // 「利用規約」のリンク先('' でリンクなし)
},
```

### 1-10. 転記・送信の挙動

```js
autoSubmit: true,        // ON(既定): チャット完了時にLPの注文ボタンを自動で押す
autoSubmit: false,       // OFF: 転記だけしてフォームへスクロール誘導(⚠️LPが確認画面スキップ設定でない場合の検討用)
hideForm: false,         // OFF(既定): LPのフォームはそのまま
hideForm: true,          // ON: LPの注文フォームを隠す(skipのダミー値を見せない。必要な場面では自動で再表示)
transferMode: 'auto',    // 既定: LP内フォームがあれば直接入力、無ければリダイレクト
transferMode: 'redirect',// 常にリダイレクト(カート型ページ用。遷移先にorders_new_autofill.html設置が必要)
ecforceOrderUrl: 'https://hugskin.shop/shop/orders/new',   // リダイレクト先
loginUrl: 'https://hugskin.shop/shop/customers/sign_in',   // メール既登録検知時のログイン画面URL
```

### 1-11. 支払い選択肢の表示

```js
paymentLabels: { 'クレジット': 'クレジットカード【手数料0円】' },
    // 支払いボタンの表示文言だけ差し替え(⚠️変更後も「クレジット」「後払い」の語は残すこと。判定に使うため)
paymentChoices: [ { label: 'クレジットカード', value: '21' } ],
    // 選択肢の手動指定(通常はLPフォームから自動生成されるので不要)
```

### 1-12. 離脱防止①: ×押下時の閉じ確認(closeConfirm)

```js
closeConfirm: {},        // ON(最小): ×を押すと「閉じますか?」確認を挟む(文言は既定)
closeConfirm: {          // ON(フル指定):
  title: '初回限定チャットを閉じますか？',
  stayText: 'チャットに戻る',
  leaveText: '閉じる',
  image: 'https://yhozumi-stack.github.io/hugskin-chatbot/img/anshin.png',   // 白部分の下のクリエイティブ('' で白部分のみ)
},
// closeConfirm を書かない = OFF(既定)
// ⚠️popup.jsの'chatclose'トリガーと併用しない(二重引き止めになる)
// 計測: hs_chat_close_confirm_show / _stay / _leave が自動で飛ぶ
```

### 1-13. 離脱防止②: 与信NGリカバリー(paymentFallback・v3.26.0〜)

```js
paymentFallback: {},     // ON(最小): 後払いの与信NGで弾き返された画面を検知→クレカ再注文に自動誘導
paymentFallback: {       // ON(文言カスタム):
  msg:   '今回のお手続きでは、後払い決済がご利用いただけませんでした🙏\n誠に申し訳ございませんが、クレジットカードでのお手続きをお願いいたします',
  msg2:  'クレジットカード情報のご入力をお願いいたします',
  pwMsg: 'セキュリティ保護のため、パスワードのみもう一度ご入力ください🙏',
},
// paymentFallback を書かない = OFF(既定)
// ⚠️ecforce側の後払いが「リアルタイム(同期)与信」設定のLPでだけ意味がある。詳細はレシピ18
```

### 1-14. 見た目・その他

```js
theme: { brand: '#C8869A', brandDark: '#a86880', brandLight: '#f9f1f4' },   // 色(バナー・吹き出し・ボタン全部連動)
title: 'お申し込みサポート',            // ヘッダーのタイトル
subtitle: 'かんたん注文チャット（約1分）',   // ヘッダーのサブタイトル
avatar: 'https://…/operator.png',      // オペレーター画像('' で💬絵文字)
typingMs: 120,                          // ボット発話前の「間」(0で完全即時)
countdown: true,                        // ON: 緊急性カウントダウンバー(毎日23:59:59まで)
countdown: { text: 'お急ぎください！本日受付終了まで' },   // ON+文言指定
mode: 'inline',                         // LP内埋め込みモード(既定は'float')
inlineSelector: '#hs-chat',             // inlineの描画先
```

### 1-15. 設定不要で常に動くもの(タグ制御なし・参考)

- **メール既登録の自動検知**: 送信後にecforceが「既に登録されています」で弾いた画面を検知→ログイン案内(全シナリオ共通)
- **カゴ落ちタグ連携(earlyLeadCapture)**: メール・電話の確定と同時にLPフォームへ即転記
- **計測イベント**(hs_chat_open / step_* / summary_view / submit / close 等)の自動送信

## 2. 離脱ポップアップ (window.HS_POPUP) — chatbot.jsとは別ファイル・チャット無しLPでも単体で動く

### 2-1. 全体ON/OFF

```js
enabled: true,    // ON(既定)
enabled: false,   // OFF: タグを消さずに一時停止
```

### 2-2. 出すタイミング(複数指定OK・どれかで1回だけ)

```js
triggers: ['back', 'delay60000'],   // 既定: ブラウザバック+60秒経過
triggers: ['back'],                 // ブラウザバックのみ
triggers: ['delay30000'],           // 30秒経過のみ(ミリ秒)
triggers: ['back', 'chatclose'],    // バック+チャットを×で閉じた瞬間(⚠️closeConfirmと併用しない)
triggers: ['back', 'visibility'],   // バック+別タブ・別アプリから戻ってきた瞬間
```

### 2-3. 表示頻度

```js
oncePer: 'session',   // 既定: タブを閉じるまで1回(LPのu=単位で記録)
oncePer: 'load',      // ページ表示ごとに1回
oncePer: 'always',    // 毎回(テスト用)
```

### 2-4. 見た目(imageの有無で2モード自動切替)

```js
/* --- 画像モード(推奨): imageを設定すると画像+重ねCTAボタンになる --- */
image: 'https://…/popup.gif',   // ON: 画像モード(GIF可。コピーは画像に焼き込む前提)
imageClickable: true,           // ON: 画像全体をCTA化(ada-cloud式・ボタン焼き込みGIF用)。既定はボタンと×だけ反応
ctaBottom: '7%',                // ボタンの画像下端からの位置
ctaWidth: '78%',                // ボタンの幅
ctaImage: 'https://…/btn.png',  // CTAボタン自体を画像にする

/* --- テキストモード: imageが空('')の時 --- */
badge: '初回限定',               // タイトル上の小ラベル('' で非表示)
title: 'ちょっとお待ちください！',
lines: [                        // 本文(1要素=1行。{{VAR}}未設定の行は自動非表示)
  'ただいま初回限定 {{PRICE}} でご案内しています。',
  'このページを閉じると、次回は通常価格でのご案内になる場合があります。',
],
```

### 2-5. ボタンの文言・飛び先

```js
ctaText: '今すぐ試してみる',    // CTAボタンの文言
ctaAction: 'chat',             // チャット起動(チャット無しLPでは自動でフォームスクロールに切替)
ctaAction: 'form',             // 注文フォームへスクロール
ctaAction: 'close',            // 閉じるだけ(「LPに戻る」ボタンとして使う)
ctaAction: 'https://…',        // URLへ遷移
closeText: '今回は見送る',      // 下の小さい閉じるリンク('' で非表示。×は常に出る)
```

### 2-6. 変数・色

```js
vars: { PRICE: '初回1,980円(税込)' },   // チャット併設LPでは書かなくてもHS_CHAT.varsを自動引き継ぎ
theme: { brand: '#C8869A', text: '#4a3b40' },
```

### 2-7. 上級者向け(通常は触らない)

```js
zIndex: 2147482900,     // チャットパネルより下(チャットが開けばポップアップより前)
formSelector: 'input[name="order[billing_address_attributes][name01]"]',   // 'form'スクロール先の検出
delayRetryMs: 10000,    // チャット操作中等で出せなかった時の再判定間隔
```

### 2-8. 設定不要で常に効くもの(参考)

- チャットを開いている間・注文送信済み・フォーム入力中は出さない(自動判定)
- backトリガーの番兵は1つだけ=2回目の「戻る」は素直に離脱させる(この設計は変えない)
- 計測: `hs_chat_popup_show / _cta / _close`(+`hs_popup_trigger`でどのトリガーか分かる)

## ⚠️ 全体の注意(タグ編集時の鉄則)

1. **タグ編集は push不要・即時・そのLPだけ**。逆に chatbot.js / popup.js の編集はpush後最大10分で**全LP**に効く
2. chatbot.js / popup.js を更新した時に即反映させたい場合は、タグの `?v=` を+1する(キャッシュ破り)
3. 凍結ルール: 本番シナリオ(formplus/standard)の中身・DEFAULTS既定値・ENGINEは書き換え禁止。実験は「別名シナリオ」か「既定OFF機能のタグON」で
4. イベント名(hs_chat_*)に依存した集計が動いているため、計測まわりの名前は変えない
5. 迷ったら CLAUDE.md の「変更判断ガイド」と各レシピを参照(このファイルは「何がタグで出来るか」の索引、手順の正本はレシピ)
